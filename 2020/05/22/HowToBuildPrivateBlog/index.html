Github + Hexo 架設個人 BLOG

發表於 2020-05-22 更新於 2020-05-25

因為想記錄自己的學習過程, 經過評估後,
決定透過 Github + Hexo 的方式來架設個人部落格; 雖然有幾點限制:

GitHub Pages source repositories have a recommended limit of 1GB.
Published GitHub Pages sites may be no larger than 1 GB.
GitHub Pages sites have a soft limit of 10 builds per hour.
GitHub Pages sites have a soft bandwidth limit of 100GB per month.

總結來說, 網站的上限不能超過 1GB, 一個小時內的網站更新不得高於 10 次以及每月流量不能大於 100GB! 但對於記錄學習筆記, 瑕不掩瑜!

一、前言

使用 Github 的靜態網頁服務, 架設個人的 BLOG, 從本地已部署的 Hexo 後台服務編輯文章, 編輯完成後再推上去即可 (如本站: https://blavince.github.io).

二、總覽

  1. 申請 Github 帳號, 創建 repository, 設定 GitHub Pages.
  2. 安裝軟體: Git、Node.js
  3. 部署本地資料夾 (用來放整個網站的資料)
  4. 本地編輯文章.
  5. 部署到 Github 上.
